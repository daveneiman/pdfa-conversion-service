Basic Usage Details:

  • Conventions Used In Examples:
    Below are the conventions we will be using within the examples given.
    /PDF/A Converter
    The context root is assumed to be /pdfa-converter-service. Your context root may be different depending on installation
    HOST
    Refers to your server where pdfa-converter-service has been installed.
    PORT
    Refers to the port on which pdfa-converter-service is running.
    REMOTEFILE
    Refers to the filename with full path on the server to convert
    LOCALFILE
    Refers to a file which you will upload to the PDF/A Converter with full path
  • Endpoint: /convert
    Convert a file and returning the corresponding PDF/A if the input file is of a type that the service is able to convert.
    GET (using curl):
    curl --get -k --data-binary file=REMOTEFILE http://HOST:PORT/pdfa-converter-service/convert
    GET (using browser):
    http://HOST:PORT/pdfa-converter-servic/convert?file=REMOTEFILE
    POST (using curl):
    curl -k -F datafile=LOCALFILE http://HOST:PORT/pdfa-converter-service/convert
    POST (using browser):
    http://HOST:PORT/pdfa-converter-servic/index.html Use the upload form.
  • Endpoint: /version
    Obtaining the version information for the PDF Converter being used to convert the input files returned in plain text format. (GET request only)
    GET (using curl):
    curl --get http://HOST:PORT/pdfa-converter-service/version
    GET (using browser)
    http://HOST:PORT/pdfa-converter-service/version
  • Information
    The return of any PDFA Converter service call will be a structured XML. If the file is properly identified this will be the PDFA Converter output, if there is an issue with the uploaded file an XML error response.
